1️⃣ EC2란 무엇인가?

EC2(Elastic Compute Cloud)는
AWS가 제공하는 가상 컴퓨터(VM) 서비스야.

간단히 말하면:

“AWS 서버실에 있는 컴퓨터를 시간 단위로 빌려서 쓰는 것이다.”

EC2로 할 수 있는 것:

웹 서버 운영 (Node.js, Spring 등)

DB 서버 구축 (MySQL, PostgreSQL)

배치 서버 / API 서버

Docker 실행

머신러닝 서버

스마트팜 센서 데이터 수집/분석 서버

VPN 서버

즉, 모든 클라우드 인프라의 시작점이 되는 서비스.

2️⃣ EC2 인스턴스 타입: T2 / T3 / T4g
✔ 공통점

모두 "T 시리즈" = 저비용/범용 버스트 가능한 컴퓨팅 인스턴스

CPU 크레딧 시스템 기반

웹 서버, 사내 툴 서버, 작은 DB 등에 적합

✔ T2 (구세대, Legacy)

CPU: 인텔 구형

성능 낮음

가격은 보통 T3보다 비쌈 (비효율적)

현재는 추천 안 함

실제 기업에서는 더 이상 T2는 거의 안 씀.
(학습용 EC2 튜토리얼에서만 가끔 보일 정도)

✔ T3 (표준적이고 가장 많이 쓰는 중급형)

CPU: 인텔 또는 AMD 기반

비용 효율 좋고 성능도 안정적

시스템 엔지니어와 백엔드 개발자들이 가장 많이 씀

대부분의 스타트업/중소기업 환경에서 기본값처럼 사용됨

추천 상황:

웹 서버 운영

REST API

백엔드 개발 테스트

소형 DB

✔ T4g (AWS Graviton 기반 ARM CPU)

AWS가 직접 설계한 고성능 ARM 기반 CPU

T3보다 성능 20~40% 증가, 가격 20% 절감

단, ARM 기반이라 이미지/라이브러리 호환 여부 필요

스마트팜 미래 계획과도 잘 맞는 이유:
ARM은 IoT/임베디드/에지 디바이스에서 많이 쓰이는 CPU라서
클라우드에서도 ARM 서버를 쓰면 얼라인이 잘 맞는다.

실제 대기업·중견 SI 업체들에서도 최근 T4g 적극 채택 중.

3️⃣ 쉽게 정리한 선택 가이드
목적	추천 인스턴스
실습 / 학습	t3.micro 또는 t4g.micro
스타트업 운영용 웹 서버	t3.medium 이상
ARM 기반(IoT/스마트팜과 연계)	t4g.micro / t4g.small
구형 튜토리얼 따라 하기	t2.micro(권장 X)
4️⃣ Smart Farm 관점에서의 EC2 선택

네가 미래에 하고 싶은 "스마트팜 설비" 관점으로 보면:

스마트 센서(온도/습도/조도/펌프) → 대부분 ARM 기반 라즈베리파이 계열

IoT 게이트웨이 → MQTT / HTTP

서버 로직 → ARM 기반도 매우 유리

그래서 장기적으로는 T4g 계열 + MQTT 브로커 + API 서버
이 조합이 아주 좋은 선택이다.
